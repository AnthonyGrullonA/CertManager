{# Autenticación #}

Autenticación

Cada petición debe incluir una clave de API en un encabezado. Acepta dos formas: 

Authorization: Api-Key cf_live_xxxxxxxxxxxxxxxxxxxx
# o, alternativamente:
X-Api-Key: cf_live_xxxxxxxxxxxxxxxxxxxx
  • cf_live_… — clave de acceso total (lectura y escritura).
  • cf_ro_… — clave de solo lectura: solo permite GET.
  • La clave hereda los permisos (grupos/ámbito) del usuario que la creó.
  • Sin clave válida, la API responde 401; con clave de solo lectura en escritura, 403.
{# Ejemplo #}

Ejemplo

# Listar certificados por vencer
curl -H "Authorization: Api-Key cf_live_xxx" \
     "{{ api_base }}/certificates/?status=POR_VENCER"

# Probar ahora (chequeo SSL inmediato) — requiere acceso total
curl -X POST -H "Authorization: Api-Key cf_live_xxx" \
     "{{ api_base }}/certificates/1/test/"

{# Errores #}

Códigos de respuesta

  • 200/201 — éxito.
  • 401 — falta la clave o es inválida/revocada.
  • 403 — la clave es de solo lectura o no tiene acceso al recurso.
  • 429 — límite de tasa (p. ej. en «Probar ahora»).
